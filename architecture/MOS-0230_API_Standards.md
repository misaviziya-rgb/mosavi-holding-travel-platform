## 2. قراردادهای نام‌گذاری (Naming Conventions) — پیشنهاد استاندارد صنعتی

- مسیرها با اسم جمع و kebab-case: `/bookings`, `/agency-partners`
- افعال HTTP برای عملیات: `GET` (خواندن)، `POST` (ایجاد)، `PUT/PATCH` (به‌روزرسانی)، `DELETE` (حذف)
- بدون فعل در مسیر (پرهیز از `/getBooking`)
- فیلدهای JSON با camelCase

## 3. نسخه‌بندی API

پیشنهاد: نسخه‌بندی در مسیر URL (`/v1/bookings`) به دلیل وضوح و سادگی کش‌گذاری.
جایگزین: نسخه‌بندی از طریق Header (`Accept-Version`). `[TBD — تأیید CSA]`

## 4. ساختار پاسخ استاندارد (پیشنهادی)

```json
{
  "data": { },
  "meta": { "requestId": "string", "timestamp": "ISO8601" },
  "errors": []
}
```

## 5. مدیریت خطا (پیشنهادی)

- استفاده از کدهای وضعیت HTTP استاندارد (400، 401، 403، 404، 409، 422، 429، 500).
- ساختار خطا:
```json
{
  "errors": [
    { "code": "VALIDATION_ERROR", "field": "email", "message": "شرح خطا" }
  ]
}
```
- کدهای خطای دامنه‌ای اختصاصی باید در پیوست هر سرویس مستند شوند.

## 6. صفحه‌بندی، فیلتر و مرتب‌سازی (پیشنهادی)

- Pagination: Cursor-based برای منابع پرحجم (مانند نتایج جستجو)، Offset-based برای منابع کوچک.
- پارامترهای استاندارد: `?page[cursor]=...&page[size]=...&sort=-createdAt&filter[status]=confirmed`
